## 📋 분석 준비 체크리스트

### 필요한 자료 (3가지 중 1가지 선택)

- [ ] **옵션 1: Figma 문서 전체 텍스트**
  - Figma 파일 내 모든 프레임/컴포넌트 명세
  - 페이지별 레이아웃, 컴포넌트, 인터랙션 정의
  - 디자인 시스템 토큰 (색상, 폰트, 간격)
  - **장점**: 가장 정확하고 자동화된 비교 가능
  - **소요 시간**: 전체 분석 약 30-45분

- [ ] **옵션 2: 주요 페이지 명세서**
  - 우선순위 높은 10-15개 페이지만 선별
  - 각 페이지별 주요 기능 목록 (버튼, 입력 필드, 탭 등)
  - 데이터 필드 및 상태 정의
  - **장점**: 빠른 분석, 핵심 이슈 집중
  - **소요 시간**: 선택적 분석 약 15-20분

- [ ] **옵션 3: 스크린샷 + 설명**
  - 주요 화면 스크린샷 (페이지당 1-2장)
  - 각 스크린샷에 대한 기능 설명 (텍스트)
  - 인터랙션 플로우 다이어그램 (있으면)
  - **장점**: 시각적으로 직관적, 준비 간단
  - **소요 시간**: 시각적 분석 약 20-30분

## 🔍 분석 방법론

### 1단계: 레이아웃 일치도 검증

**검증 항목**:
- [ ] 화면 구조 (헤더/본문/푸터 배치)
- [ ] 컴포넌트 배치 순서
- [ ] 그리드/플렉스 레이아웃 구조
- [ ] 반응형 브레이크포인트

**채점 기준**:
- ✅ **완전 일치** (100%): 디자인과 구현이 정확히 동일
- 🟢 **거의 일치** (80-99%): 미세한 간격 차이만 존재
- 🟡 **부분 일치** (50-79%): 주요 구조는 같으나 세부 배치 다름
- 🔴 **불일치** (0-49%): 완전히 다른 레이아웃 또는 미구현

### 2단계: 기능 완성도 검증

**검증 항목**:
- [ ] 모든 버튼이 디자인과 동일하게 존재하는가?
- [ ] 입력 필드 (텍스트, 선택, 날짜 등) 모두 구현되었는가?
- [ ] 탭/네비게이션 구조가 일치하는가?
- [ ] 모달/팝업/드롭다운이 디자인과 일치하는가?
- [ ] 에러/로딩/빈 상태 화면이 구현되었는가?

**채점 기준**:
- ✅ **완전 구현** (100%): 모든 기능이 동작함
- 🟢 **대부분 구현** (80-99%): 1-2개 사소한 기능만 누락
- 🟡 **부분 구현** (50-79%): 주요 기능은 있지만 일부 누락
- 🔴 **미구현** (0-49%): 핵심 기능이 누락됨

### 3단계: 콘텐츠 정확도 검증

**검증 항목**:
- [ ] 라벨/버튼 텍스트가 디자인과 일치하는가?
- [ ] 데이터 필드 (이름, 날짜, 상태 등)가 올바르게 표시되는가?
- [ ] 플레이스홀더 텍스트가 일치하는가?
- [ ] 에러 메시지가 디자인 가이드를 따르는가?

**채점 기준**:
- ✅ **완전 일치** (100%)
- 🟢 **거의 일치** (80-99%): 1-2개 텍스트만 다름
- 🟡 **부분 일치** (50-79%): 여러 텍스트/라벨이 다름
- 🔴 **불일치** (0-49%): 대부분의 텍스트가 다름

### 4단계: 인터랙션 일치 검증

**검증 항목**:
- [ ] 버튼 클릭 시 동작이 디자인 명세와 일치하는가?
- [ ] 스와이프/드래그 제스처가 구현되었는가?
- [ ] 애니메이션/전환 효과가 있는가?
- [ ] 상태 변화 (hover, active, disabled)가 구현되었는가?

**채점 기준**:
- ✅ **완전 일치** (100%)
- 🟢 **거의 일치** (80-99%)
- 🟡 **부분 일치** (50-79%)
- 🔴 **불일치** (0-49%)

### 5단계: 스타일 매칭 검증

**검증 항목**:
- [ ] 색상 (브랜드 컬러, 배경, 텍스트)
- [ ] 폰트 (크기, 굵기, 줄 높이)
- [ ] 간격 (패딩, 마진, 간격)
- [ ] 아이콘 (크기, 스타일, 색상)
- [ ] 그림자/보더/라운딩

**채점 기준**:
- ✅ **완전 일치** (100%)
- 🟢 **거의 일치** (80-99%): 미세한 차이만
- 🟡 **부분 일치** (50-79%): 여러 요소가 다름
- 🔴 **불일치** (0-49%): 디자인 시스템 미적용

## 📝 분석 시작 가이드

### Step 1: 자료 수집
1. Figma 디자인 파일 또는 명세서 준비
2. 현재 구현 코드 최신 버전 확인 (`git pull`)
3. 로컬 개발 서버 실행 (`npm start`)

### Step 2: 분석 실행
Figma 자료를 아래 형식 중 하나로 제공:

**형식 1: Figma 문서 텍스트**
```
페이지: ProfilePage
레이아웃:
  - 헤더: 뒤로가기 버튼, "프로필", 설정 버튼
  - 본문:
    - 프로필 카드: 아바타, 이름, 이메일, "프로필 수정" 버튼
    - 스킬 섹션: "내 스킬", 스킬 태그 목록, "편집" 버튼
    - 프로젝트 섹션: "참여한 프로젝트", 프로젝트 카드 목록
...
```

**형식 2: 주요 페이지 명세**
```markdown
# ProfilePage 기능 명세

## 필수 기능
- [ ] 프로필 조회
- [ ] 프로필 편집 버튼
- [ ] 아바타 변경
- [ ] 스킬 편집 기능
- [ ] 참여 프로젝트 목록 (완료/진행중 구분)

## 데이터 필드
- 이름 (name)
- 이메일 (email)
- 자기소개 (bio) - 최대 500자
- 스킬 (skills) - 배열, 최대 10개
...
```

**형식 3: 스크린샷 + 설명**
```
[ProfilePage 스크린샷 업로드]
설명:
- 상단에 헤더 (뒤로가기, 제목, 설정)
- 프로필 카드에 "프로필 수정" 버튼 있음
- 스킬 섹션에 "편집" 버튼 있음
- 하단에 참여 프로젝트 목록
```

### Step 3: 결과 검토
1. 생성된 분석 결과 검토
2. 우선순위 확인 및 조정
3. 개발 스프린트에 반영
